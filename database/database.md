## 데이터베이스 상세 설명


### 1. users

유저를 기록하기 위하여 존재하며, 학생, 교사 및 관리자가 모두 포함된다.\
각각의 유저는 UUID를 가지며, 해당 UUID는 RFC 4122에 따른 UUID v4를 기반으로 생성한다.\
학생 계정의 대량 생성에 대비하여 pw를 임의의 값(bssm2026!)을 기본값으로 설정하였으며, 해당 값은 변경 될 수 있다.\
role은 단순 구분 용도로 아래의 기준표에 따라서 분류된다.


| 번호 | 구분   |
| ---- | ------ |
| 0    | 학생   |
| 1    | 교사   |
| 2    | 관리자 |

#### 설계 사유

학생과 교사 테이블에 ID / PW를 각각 기입 할 경우 교사와 학생간 ID 충돌 등의 위험이 있으므로, 공통된 유저 테이블을 생성하고, 학생과 교사는 각각의 테이블에 0 : 1 관계로써 연결되게 하여 유연한 대처가 가능하도록 제작하였다.

### 2. students

학생의 경우 해당 테이블에 학생 정보가 기록된다.\
성명, 학년, 학번, 번호가 각각 기입되며 자퇴/전학 등의 사유로 학적 상태가 변동될 가능성을 고려하여 status 컬럼이 존재한다. status는 아래의 기준표에 따라 분류된다.

| 번호 | 구분 |
| ---- | ---- |
| 0    | 재학 |
| 1    | 자퇴 |
| 2    | 전학 |
| 3    | 퇴학 |
| 4    | 졸업 |
